# Masked Face Verification Spike

This repository contains a small Colab-first validation spike for masked face
verification. The goal is to test whether an occlusion-aware upper-face variant
is worth pursuing before investing in a larger project.

## Deliverables

- `notebooks/validation_spike.ipynb` - runnable notebook for the first pass.
- `results/validation_results.csv` - compact metrics table generated by the
  notebook.
- `results/validation_conclusion.md` - generated go/no-go recommendation.
- `results/qualitative_examples/` - generated examples where methods differ.

## Dataset

Use a real masked-recognition dataset with both masked and unmasked images per
identity. The preferred layout is one identity per folder:

```text
data/pku_masked_face_subset/
  person_001/
    masked/
      *.jpg
    unmasked/
      *.jpg
  person_002/
    masked/
    unmasked/
```

The notebook also accepts common RMFRD/SMFRD-style folders and can normalize
them into this layout. Start with roughly 100-200 identities.

## Model Variants

The notebook evaluates:

- Baseline FaceNet on the full detected face.
- Upper-face FaceNet, which crops/resizes the non-occluded upper face.
- A masked-specific candidate slot. It runs with a CASIA-WebFace FaceNet
  fallback by default so the notebook is executable, but the intended checkpoint
  is a masked-recognition model such as MaskInv KD / ElasticFace-Arc from
  `fdbtrs/Masked-Face-Recognition-KD`.

## Colab

Open `notebooks/validation_spike.ipynb` in Colab, run the setup cells, mount
Drive or upload the dataset archive, and then run all cells. The notebook saves
all outputs under `results/`.

The project follows Colab's runtime dependencies. Do not run a broad
`pip install -r requirements.txt` before the benchmark; that can replace
Colab's preinstalled `torch`, `numpy`, and related packages. Use the included
installer when the notebook needs optional packages:

```python
!python scripts/install_colab_deps.py
```

For a reliable MCP path, use a tiny bootstrap cell instead of pasting the
notebook through the browser. The smoke path does not need the full notebook
dependencies:

```python
!git clone https://github.com/tungd/masked-face-id.git /content/masked-face-id
%cd /content/masked-face-id
!python scripts/run_validation_spike.py --smoke
```

After mounting a real dataset at `/content/pku_masked_face_subset`, run the
notebook for the full FaceNet-based benchmark.
